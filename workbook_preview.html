Design Rationale

The old layout had Positions (input) and Attribution (output) as separate sheets, requiring cross-sheet lookups. The new design merges them into a single Monitor sheet so each bond is one row with everything visible.

Column Grouping Principle

For each risk factor, the three related columns sit side-by-side:

Sensitivity
×
Factor Change
=
Attribution

This lets the reviewer instantly verify: "is delta × equity move = equity attribution?" without jumping between sheets.

Adjustment Logic (X + Spread columns)
X = parallel shift — moves BOTH bid and ask equally (e.g. +0.50 lifts both)
X_Bid = additional shift to bid only
X_Ask = additional shift to ask only
Spread = symmetric widening — subtracted from bid, added to ask

Adj_Bid = Curr_Bid + X + X_Bid − Spread
Adj_Ask = Curr_Ask + X + X_Ask + Spread
Adj_Mid = (Adj_Bid + Adj_Ask) / 2   ← Spread cancels in mid (−S and +S net to 0)
Move_Mid = Adj_Mid − Prev_Mid   ← this is the move used for attribution

BDP Formula Pattern (all cells use IFERROR wrapper)
=IFERROR(BDP("/isin/"&A2&" Corp","BID"),"")    ← BGN (no @source)
=IFERROR(BDP("/isin/"&A2&"@BVAL Corp","BID"),"")    ← BVAL
=IFERROR(BDP("/isin/"&A2&"@CBBT Corp","BID"),"")    ← CBBT (executable)

Best Bid / Ask / Mid
Best_Bid = MAX(BGN_Bid, BVAL_Bid, CBBT_Bid, QMGR_Bid, LCPR_Bid, NOMX_Bid, ...)
Best_Ask = MIN(BGN_Ask, BVAL_Ask, CBBT_Ask, QMGR_Ask, LCPR_Ask, NOMX_Ask, ...)
Best_Mid = IF(AND(Best_Bid<>"", Best_Ask<>""), (Best_Bid+Best_Ask)/2, "")
QMGR Staleness Check (column L)
=IF(IFERROR(TODAY()-DATEVALUE(BDP("/isin/"&A2&"@QMGR Corp","LAST_UPDATE_DT_EXCH_TZ")),99)>1, "STALE", "FRESH")
APAC CB tip: QMGR is the primary quote source for Asian convertible bonds — dealers distribute runs via Bloomberg IB messages rather than electronic platforms. Always check QM_Time; if blank, the dealer hasn't sent a run recently. Use CBBT if available, or call the desk. The BID_SIZE_TDY / ASK_SIZE_TDY fields on QMGR give you quoted size, which matters for large CB positions.
